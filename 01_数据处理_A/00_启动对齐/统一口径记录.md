## 5. 统一单位与符号口径

| 项目 | 统一口径 |
| ---- | -------- |
| 透水率 | 使用原始数值，正文中统一称“透水率值” |
| 时间戳 | 使用原始检测时间 |
| 趋势建模时间变量 | 优先用“天”作为连续时间单位 |
| 维护间隔 | 统一用“天” |
| 寿命 | 统一用“年”表示 |
| 成本 | 统一用“万元”表示 |

数学符号方面：

- 透水率记为 `$P_i(t)$`
- 维护恢复量记为 `$\Delta P$`
- 年均成本记为 `$C$`
- 中维护次数记为 `$N_m$`
- 大维护次数记为 `$N_l$`

## 6. 统一数据处理规则

### 6.1 时间基准

- 统一观测起点暂定为全部数据中最早的有效时间：`2024-04-03 01:00:05`
- `day_index` 定义为样本时间与统一观测起点的天数差
- `hour_index` 定义为样本时间与统一观测起点的小时数差

### 6.2 季节划分

| 季节 | 月份范围 |
| ---- | -------- |
| 春 | 3—5 月 |
| 夏 | 6—8 月 |
| 秋 | 9—11 月 |
| 冬 | 12—2 月 |

### 6.3 缺失值处理

- 主清洗表保留所有原始记录，不在总表中直接删除缺失行
- 对缺失的 `per` 值统一保留为空，并标记 `is_missing=1`
- 图表或模型中若需插值、平滑或聚合，必须在派生表中单独进行，并记录处理方法
- 不允许在没有说明的情况下直接用均值填补缺失值

### 6.4 采样频率处理

- 原始主表保留原采样粒度
- 为了支持趋势和季节分析，A 额外输出一份“按日聚合表”
- 需要统一比较不同设备趋势时，优先使用日均透水率或周均透水率，而不是直接混用原始不规则高频数据

### 6.5 维护前后匹配规则

- `per_before`：取维护日期前 72 小时内最近一次有效检测值
- `per_after`：取维护日期后 72 小时内最近一次有效检测值
- 若 72 小时窗口内无有效值，则该条记录在维护匹配表中标记为“缺测”
- `avg_after_3d`、`avg_after_7d`、`avg_after_15d` 分别表示维护后对应窗口内的平均透水率

## 7. 统一寿命判据口径

依据题面，先采用以下统一暂定口径，供 B 在第 2 问建模时细化：

1. 当年平均透水率低于 37；
2. 且最近一次维护后恢复量显著不足。

为了便于后续计算，启动阶段先约定“恢复量显著不足”默认表示：

- 当前维护恢复量低于历史同类维护平均恢复量的 20%

说明：

- 这是阶段 0 的统一暂定口径，不是最终定论
- 若 B 在第 2 问验证后认为应改为 30% 或改为分位数阈值，需要在模型说明中显式更新
