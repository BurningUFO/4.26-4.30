# A 侧交付物说明

本目录用于存放成员 A 的结构化输出、阶段统计表和后续可直接交接给 B、C 的结果文件。

## 阶段 1 交付物

### Markdown 说明文件

- `数据结构说明表.md`
  - 作用：汇总附件 1 和附件 2 的字段结构、记录规模、日期范围、重命名建议和可直接使用性判断。
- `维护记录概览表.md`
  - 作用：汇总每台过滤器的中维护/大维护次数、维护日期范围和维护匹配可行性。
- `异常情况清单.md`
  - 作用：登记缺失值、异常值、长时间断档、编号不一致和维护匹配缺口。

### CSV 统计文件

- `阶段1_过滤器盘点统计表.csv`
  - 作用：提供每台过滤器的记录数、透水率缺失值、重复值、异常值、时间跨度和断档统计。
- `阶段1_透水率异常值统计表.csv`
  - 作用：提供按 IQR 规则初筛的异常值区间和异常数量。
- `阶段1_维护记录统计表.csv`
  - 作用：提供每台过滤器的维护次数、日期范围和维护间隔统计。
- `阶段1_维护匹配可行性检查表.csv`
  - 作用：检查维护记录是否能在 `24h/72h/168h` 窗口下匹配到维护前后有效透水率。

## 阶段 2 交付物

### Excel 数据文件

- `clean_data.xlsx`
  - 作用：作为成员 A 的透水率清洗总表，供阶段 3 分析和后续 B 建模直接读取。
  - 主要字段：`filter_id`、`time`、`date`、`per`、`day_index`、`hour_index`、`year`、`month`、`season`、`source_sheet`、`is_missing`、`is_outlier`。
- `maintenance_record.xlsx`
  - 作用：作为清洗后的维护记录表，供维护匹配、维护效果统计和 B 建模直接读取。
  - 主要字段：`filter_id`、`date`、`maintain_type`、`maintain_level`、`year`、`month`、`season`、`source_sheet`。

### Markdown 说明文件

- `数据清洗说明.md`
  - 作用：说明阶段 2 的清洗口径、字段含义、保留规则、统计结果和复现方式。

## 阶段 3 交付物

### Excel 数据文件

- `maintenance_match.xlsx`
  - 作用：保存维护记录与透水率数据的标准匹配结果。
  - 主要字段：`before_per`、`after_per`、`delta_per`、`relative_recovery_rate`、`after_3d_mean`、`after_7d_mean`、`after_15d_mean`、`post_15d_decline_rate`、`maintenance_effectiveness_coef`、`remark`。
- `维护效果统计表.xlsx`
  - 作用：按维护类型和过滤器汇总恢复量、恢复率、维护后衰减率和有效性系数。
  - 主要工作表：`summary_by_type`、`summary_by_filter_type`、`decay_summary`。
- `每台过滤器下降率表.xlsx`
  - 作用：保存第 1 问趋势和季节性分析的核心参数。
  - 主要工作表：`decline_rate_by_filter`、`monthly_average_overall`、`seasonal_average_overall`、`monthly_average_by_filter`、`seasonal_average_by_filter`。
- `异常跳变点联动分析表.xlsx`
  - 作用：保存异常点、跳变点与维护记录之间的联动分析结果。
  - 主要工作表：`jump_points`、`jump_thresholds`、`jump_summary_by_filter`、`jump_relation_summary`、`maintenance_jump_summary`、`outlier_days`、`outlier_summary_by_filter`。

### Markdown 说明文件

- `第1问结论要点.md`
  - 作用：汇总第 1 问可直接交给 B 和 C 的文字结论与指标解释。
- `异常点与跳变点分析说明.md`
  - 作用：说明缺失值、IQR 异常值、断档和日尺度跳变点的识别与处理口径，回应 C 侧关于异常处理过于简单的反馈。
- `维护记录与透水率联动分析.md`
  - 作用：说明透水率监测数据与维护记录的事件联动关系，回应 C 侧关于数据内在联系不足的反馈。
- `C反馈回应与增强说明.md`
  - 作用：汇总 C 侧反馈要点、A 侧补强动作、新增文件和论文写作建议。
- `A_阶段0-3纵览检查与优化记录.md`
  - 作用：记录成员 A 阶段 0 至阶段 3 的完整性、合理性、图表质量和剩余风险检查。

## 阶段 4 首版交接材料

- `A_to_B_数据与建模参数交接说明.md`
  - 作用：面向成员 B 汇总可直接建模的数据文件、字段说明、数据范围、主要发现、下降率参数、维护恢复参数、季节影响系数和异常处理建议。
  - 使用对象：成员 B。
  - 后续维护：若 B 反馈需要新增字段、调整异常处理口径或补充模型输入，应在该文件中追加修订说明。

## 阶段 5 首版交接材料

- `A_to_C_第1问论文素材交接说明.md`
  - 作用：面向成员 C 汇总第 1 问题目要求、图表清单、图注建议、表格清单、论文语言结论段落和写作建议。
  - 使用对象：成员 C。
  - 后续维护：若 C 反馈需要重编号、改图注或调整正文表述，应在该文件中追加修订说明。

## 复现方式

- 入口脚本：`04_代码/A_data_process.py`
- 运行命令：`python 04_代码/A_data_process.py`
- 当前脚本功能：完成阶段 1、阶段 2、阶段 3，并覆盖生成本目录中的相关交付物。
- 阶段 4、阶段 5 交接材料为基于当前 A 侧输出整理的 Markdown 文档，不由脚本自动覆盖生成。

## 使用说明

- 阶段 1 文件主要服务于阶段 2 清洗前的口径确认，不直接替代清洗总表。
- 阶段 2 文件已经可以作为成员 B 和阶段 3 分析的标准输入。
- 阶段 3 文件已经具备向成员 B 提供建模参数、向成员 C 提供图表结论的基础。
- 阶段 4、阶段 5 文件是首版交接说明，后续按 B/C 反馈继续修订，不删除阶段 1、2、3 记录。
