# 一、总体分工原则

本题围绕过滤器透水率变化、寿命预测和维护优化展开，核心目标是建立模型，使工厂污水净化的年均成本最小。题目要求依次完成数据规律分析、寿命预测、维护优化和成本波动分析。

建议三人定位如下：

| 成员 | 角色           | 主要职责                                          |
| ---- | -------------- | ------------------------------------------------- |
| A    | 数据分析负责人 | 负责数据清洗、第1问、透水率规律分析、维护效果分析 |
| B    | 建模优化负责人 | 负责第2问寿命预测、第3问维护优化、第4问敏感性分析 |
| C    | 论文统稿负责人 | 负责论文框架、文字撰写、图表排版、摘要、最终整合  |

核心关系是：

```text
A 输出数据规律和维护效果参数
        ↓
B 基于 A 的结果建立预测和优化模型
        ↓
C 将 A、B 的结果整理成完整论文
```

------

# 二、成员A：数据分析与第1问负责人

## 1. A的核心任务

A负责把附件数据变成可用于建模的数据，并完成第1问的主要分析。

第1问要求分析：

- 透水率数据的周期性；
- 透水率下降趋势；
- 不同过滤器之间的规律差异；
- 中维护、大维护对透水率的影响；
- 能够反映透水率变化的影响指标。

## 2. A的具体工作

### 任务1：整理附件1透水率数据

将10台过滤器的数据合并成一张总表。

建议字段如下：

| 字段名    | 含义                  |
| --------- | --------------------- |
| filter_id | 过滤器编号，如 A1、A2 |
| date      | 检测日期              |
| per       | 透水率                |
| day_index | 距观测起点的天数      |
| month     | 月份                  |
| season    | 季节                  |
| year      | 年份                  |

### 任务2：整理附件2维护记录

将维护记录统一成：

| 字段名         | 含义                     |
| -------------- | ------------------------ |
| filter_id      | 过滤器编号               |
| date           | 维护日期                 |
| maintain_type  | 维护类型，中维护或大维护 |
| maintain_level | 维护等级，可设为1或2     |

### 任务3：匹配维护记录与透水率数据

A需要判断每次维护前后透水率变化，例如：

- 维护前最近一次检测透水率；
- 维护后最近一次检测透水率；
- 维护后3天、7天、15天内平均透水率；
- 维护前后透水率差值。

维护效果指标可以定义为：

$$
\Delta P = P_{\text{after}} - P_{\text{before}}
$$

其中，$\Delta P$ 表示维护带来的透水率提升量。

## 2. B负责第2问：寿命预测模型

### 建模目标

根据历史透水率数据，预测10台过滤器在当前固定维护规律下的寿命。

### 推荐模型

建议采用“趋势项 + 季节项 + 维护项”的模型：

$$
P_i(t) = \alpha_i + \beta_i t + S(t) + M(t) + \varepsilon_i(t)
$$

其中：

| 符号               | 含义                               |
| ------------------ | ---------------------------------- |
| $P_i(t)$           | 第 $i$ 台过滤器在 $t$ 时刻的透水率 |
| $\alpha_i$         | 第 $i$ 台过滤器的基础性能          |
| $\beta_i t$        | 长期下降趋势                       |
| $S(t)$             | 季节性影响                         |
| $M(t)$             | 维护影响                           |
| $\varepsilon_i(t)$ | 随机误差                           |

季节项可以写为：

$$
S(t) = a \sin \frac{2\pi t}{365} + b \cos \frac{2\pi t}{365}
$$

维护项可以写为：

$$
M(t) = \gamma_1 I_{\text{中维护}} + \gamma_2 I_{\text{大维护}}
$$

其中：

| 参数       | 含义                       |
| ---------- | -------------------------- |
| $\gamma_1$ | 中维护带来的平均透水率提升 |
| $\gamma_2$ | 大维护带来的平均透水率提升 |

### 寿命判定标准

题目中认为，当年平均透水率低于37，且维护后也无法有效改善时，过滤器寿命终止。

所以B可以设置两个条件：

1. 年平均透水率低于37；
2. 维护恢复量低于设定阈值，例如低于历史平均维护恢复量的20%或30%。

当两个条件同时满足时，判定寿命终止。

### B需要输出的结果

| 结果                 | 用途           |
| -------------------- | -------------- |
| 模型公式             | 论文第2问      |
| 参数估计表           | 支撑模型可信度 |
| 拟合优度指标         | 说明模型效果   |
| 10台过滤器预测曲线   | 展示未来变化   |
| 10台过滤器寿命预测表 | 第2问核心结果  |

寿命预测表建议格式：

| 过滤器编号 | 年均下降率 | 当前平均透水率 | 预测寿命终止时间 | 剩余寿命 |
| ---------- | ---------- | -------------- | ---------------- | -------- |
| A1         |            |                |                  |          |
| A2         |            |                |                  |          |
| ...        |            |                |                  |          |

## 3. B负责第3问：最优维护方案

### 建模目标

在满足透水率要求的前提下，使长期年均成本最小。

### 成本函数

题目给出：

- 购买一台过滤器约300万元；
- 一次中维护约3万元；
- 一次大维护约12万元；
- 小维护成本可忽略。

因此可以建立年均成本函数：

$$
C = \frac{300 + 3N_m + 12N_l}{T}
$$

其中：

| 符号  | 含义                   |
| ----- | ---------------------- |
| $C$   | 年均成本，单位万元/年  |
| $N_m$ | 生命周期内中维护次数   |
| $N_l$ | 生命周期内大维护次数   |
| $T$   | 过滤器使用寿命，单位年 |

### 优化变量

B可以设置以下决策变量：

| 变量     | 含义           |
| -------- | -------------- |
| $x_m$    | 中维护间隔天数 |
| $x_l$    | 每年大维护次数 |
| $T$      | 更换时间       |
| $P_i(t)$ | 预测透水率状态 |

### 可比较的维护策略

建议B至少比较四类方案：

| 方案     | 含义                         |
| -------- | ---------------------------- |
| 当前方案 | 按原维护规律延续             |
| 节约方案 | 减少中、大维护次数           |
| 保守方案 | 增加维护频率，延长寿命       |
| 优化方案 | 搜索使年均成本最低的维护组合 |

例如可以设定：

- 中维护间隔：30天、45天、60天、75天、90天；
- 大维护次数：每年1次、2次、3次、4次；
- 更换标准：年平均透水率低于37且维护恢复不足。

然后通过遍历或仿真，找出成本最低的组合。

### B需要输出的结果

| 结果               | 用途           |
| ------------------ | -------------- |
| 优化目标函数       | 论文第3问核心  |
| 约束条件           | 说明方案合理性 |
| 不同维护方案成本表 | 比较方案优劣   |
| 最优维护方案       | 第3问最终答案  |
| 年均成本最低值     | 量化结果       |

维护方案表建议格式：

| 方案     | 中维护间隔 | 大维护次数/年 | 预测寿命 | 年均成本 | 是否可行 |
| -------- | ---------- | ------------- | -------- | -------- | -------- |
| 当前方案 |            |               |          |          |          |
| 节约方案 |            |               |          |          |          |
| 保守方案 |            |               |          |          |          |
| 优化方案 |            |               |          |          |          |

## 4. B负责第4问：成本波动分析

### 分析目标

判断当过滤器购买价格或维护成本发生变化时，原维护方案是否仍然可行。

### 建议波动范围

| 参数           | 基准值  | 波动范围         |
| -------------- | ------- | ---------------- |
| 过滤器购买价格 | 300万元 | ±10%、±20%、±30% |
| 中维护成本     | 3万元   | ±10%、±20%、±30% |
| 大维护成本     | 12万元  | ±10%、±20%、±30% |

### 需要分析的结论

B要判断：

- 购买价格升高时，是否应该更倾向于维护而不是更换；
- 购买价格降低时，是否可以适当提前更换；
- 中维护成本升高时，中维护频率是否需要降低；
- 大维护成本升高时，是否应减少大维护次数；
- 成本波动后，原最优方案是否仍然接近最优。

### B需要输出的结果

| 结果                   | 用途             |
| ---------------------- | ---------------- |
| 成本波动敏感性表       | 第4问核心        |
| 参数变化下的最优方案表 | 判断方案是否变化 |
| 敏感性分析图           | 展示年均成本变化 |
| 稳健性结论             | 论文结论部分     |

# 八、A、B、C之间的信息交接格式

建议每次交接不要只发代码或截图，而是按固定格式交接。

## A给B的格式

```text
1. 数据文件：clean_data.xlsx
2. 字段说明：filter_id, date, per, day_index, month, season
3. 主要发现：
   - 透水率整体呈下降趋势；
   - 存在明显季节性波动；
   - 大维护平均恢复量高于中维护；
   - 不同过滤器下降速度不同。
4. 可用于建模的参数：
   - 每台过滤器平均下降率；
   - 中维护平均恢复量；
   - 大维护平均恢复量；
   - 各月份季节系数。
```

## B给C的格式

```text
1. 模型名称：趋势-季节-维护综合预测模型
2. 解决问题：预测过滤器寿命
3. 输入变量：时间、过滤器编号、季节、维护类型
4. 输出变量：预测透水率、寿命终止时间
5. 主要公式：……
6. 主要结果：……
7. 对应图表：图5、图6、表4
8. 论文中需要强调的结论：……
```

## C反馈给A、B的格式

```text
1. 哪些图表缺少单位？
2. 哪些公式缺少变量解释？
3. 哪些结果和题目问题没有对应？
4. 哪些结论需要补充数据支撑？
5. 哪些地方需要重新计算或统一口径？
```

# 十二、方案落地任务阶段流程表

为了让 `方案.md` 不只是“分工说明”，还能够直接落地执行，建议按下面的流程推进。每一阶段都要有明确输入、动作、输出和验收标志。

| 阶段 | 时间建议 | 牵头人 | 输入 | 核心动作 | 阶段输出 | 验收标志 |
| ---- | -------- | ------ | ---- | -------- | -------- | -------- |
| 0. 启动对齐 | 第0—0.5小时 | A | 题目、附件、论文规范 | 三人统一变量命名、单位口径、寿命判据和图表清单 | 统一口径记录、任务拆分表 | A/B/C都知道自己本阶段交什么 |
| 1. 数据盘点 | 第0.5—1小时 | A | `附件1.xlsx`、`附件2.xlsx` | 统计每台过滤器数据范围、缺失情况、维护次数和维护类型 | 数据结构说明表、异常情况清单 | 明确哪些数据可直接用、哪些要清洗 |
| 2. 数据清洗 | 第1—4小时 | A | 原始附件 | 合并10台过滤器数据、统一时间字段、补充 `filter_id`、`month`、`season`、`day_index` 等字段 | `clean_data.xlsx`、`maintenance_record.xlsx` | B可直接读取并建模，字段无歧义 |
| 3. 第1问分析 | 第4—10小时 | A | 清洗后数据、维护记录 | 做趋势分析、季节性分析、维护前后对比，形成维护恢复量与下降率指标 | 趋势图、季节图、维护效果表、第1问结论 | 能回答“周期性、下降趋势、维护影响、影响指标”四部分 |
| 4. 第2问建模 | 第8—16小时 | B | A交付的清洗数据和指标 | 建立趋势项、季节项、维护项综合模型，拟合参数并预测寿命 | 寿命预测公式、参数表、预测曲线、寿命表 | 10台过滤器均有寿命预测结果，且判据清晰 |
| 5. 第3问优化 | 第16—24小时 | B | 第2问预测模型、维护成本参数 | 设定维护策略组合，比较不同中维护间隔和大维护频次下的寿命与成本 | 方案比较表、最优维护方案、年均成本结果 | 能明确回答“哪种方案成本最低、为什么可行” |
| 6. 第4问敏感性分析 | 第24—30小时 | B | 最优方案、价格与维护成本基准值 | 对购买价格、中维护成本、大维护成本做波动分析 | 敏感性分析表、稳健性结论、敏感性图 | 能判断原方案在成本波动下是否仍接近最优 |
| 7. 论文整合 | 第10—32小时并行推进 | C | A/B阶段性结果 | 写问题重述、模型假设、符号说明，并同步把图表和结果写进正文 | 论文初稿、图表编号表、符号统一表 | 正文与表图能一一对应，逻辑不断层 |
| 8. 联检查错 | 第32—36小时 | C主导，A/B配合 | 论文初稿、代码、结果表 | 核对公式、单位、图表编号、结果一致性和题目对应关系 | 修改清单、终稿前确认版 | 每一问都有结论，数据与论文描述一致 |
| 9. 最终提交 | 最后阶段 | C | 终稿前确认版 | 完成摘要、格式规范检查、附录整理和导出 | 论文终稿、附录、提交包 | 文件命名正确、排版合规、可直接提交 |

## 阶段交接要求

| 交接方向 | 必交内容 | 说明 |
| -------- | -------- | ---- |
| A → B | 清洗数据表、维护匹配表、下降率表、维护恢复量表 | B不能只拿图片，必须拿到结构化表格 |
| A → C | 第1问图表、图注建议、第1问结论要点 | 方便C直接写正文和图表说明 |
| B → C | 模型公式、参数表、寿命预测表、最优方案表、敏感性结果 | C据此写第2—4问正文 |
| C → A/B | 缺口清单、返工意见、格式统一要求 | 避免最后阶段集中返工 |

## 每阶段完成后的检查问题

1. 这一阶段的输出，是否能直接被下一阶段使用？
2. 图表、表格、公式，是否已经具备论文可用格式？
3. 当前结论，是否真的对应题目中的某一问？
4. 结果是否有可复现的数据来源或代码来源？
5. 如果现在停下，C能否把已有内容写进论文？
